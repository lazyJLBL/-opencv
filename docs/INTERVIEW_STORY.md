# 智能驾驶岗位面试讲解模板

## 1. 项目定位
这是一个从传统 OpenCV 车道线检测升级而来的“计算几何驱动感知项目”，核心目标是把像素级车道线提取升级为可解释的几何状态估计：
- 消失点（Vanishing Point）
- 车道宽度（像素域 + 米制）
- 车道中心曲率半径（米制）
- 车辆横向偏移（米制）

## 2. 我主导的技术升级
1. 从单脚本改为模块化架构：geometry / perception / visualization / tools / tests。
2. 引入齐次坐标几何：直线交点和线到点距离，为消失点和约束判断提供统一表达。
3. 增加相机标定与去畸变：使用棋盘格标定相机内参与畸变参数。
4. 标定阶段自动输出质量报告：重投影误差分布、坏帧剔除、误差可视化图。
5. 用相机内参 + 安装高度/俯仰角，把像素点投影到地平面，替代经验尺度。
6. 车道模型从直线升级为二次曲线，支持曲线道路。
7. 变道检测从单阈值升级为时序状态机(FSM)+置信度评分，更适合复杂车流。
8. 加入夜晚/雨天/逆光失效检测，触发降级策略并复用历史稳定状态。
9. 建立评估闭环：支持合成数据基准和真实视频统计输出。

## 3. 面试可强调的工程点
1. 可解释性：每一帧都有几何指标，不是只画线。
2. 物理一致性：指标来自真实几何投影，不依赖固定经验比例。
3. 可验证性：标定质量报告可解释“哪些标定图被剔除、为什么剔除”。
4. 鲁棒性：颜色 + 梯度联合特征，曲线拟合 + FSM + 失效降级。
5. 可扩展性：后续可以无缝接入语义分割、卡尔曼滤波、多目标跟踪。
4. 工程完整度：CLI、批量评估、单元测试、演示看板齐全。

## 4. 你可以主动说的下一步迭代
1. 车道线从直线模型升级到二次曲线模型，并做变道场景验证。
2. 引入相机标定参数，用真实内参替代经验尺度。
3. 引入时序状态估计器（EKF/UKF）替代简单 EMA。
4. 增加失效检测和降级策略（阴影、雨夜、强反光场景）。
